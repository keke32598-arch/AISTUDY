# 检索策略

## 什么是检索策略

检索策略是指在 RAG 系统中，如何从知识库中检索与用户查询相关的文档片段的方法和技术。好的检索策略可以提高 RAG 系统的准确性和效率。

## 基本检索策略

### 1. 相似度搜索

- **基于向量相似度**：使用余弦相似度、欧氏距离等度量文本相似性
- **k-NN 搜索**：返回与查询向量最相似的 k 个文档片段
- **适用场景**：通用问答、语义搜索

### 2. 混合检索

- **关键词 + 向量混合**：结合传统关键词搜索和向量相似度搜索
- **优势**：既利用关键词的精确匹配，又利用向量的语义理解
- **实现方式**：
  - 并行检索：同时进行关键词和向量搜索，合并结果
  - 串行检索：先关键词过滤，再向量排序

### 3. 重排序（Reranking）

- **目的**：对初始检索结果进行重新排序，提高相关性
- **方法**：
  - 交叉编码器（Cross-Encoder）：同时考虑查询和文档的相关性
  - 基于规则的重排序：根据元数据、位置等因素调整排序
  - 集成学习：结合多种排序信号

## 高级检索策略

### 1. 多向量检索

- **方法**：为每个文档生成多个向量（如标题向量、内容向量）
- **优势**：捕捉文档的不同方面，提高检索准确性
- **实现**：使用不同的 Embedding 模型或不同的文本表示

### 2. 分层检索

- **方法**：先进行粗粒度检索，再进行细粒度检索
- **优势**：提高检索速度和准确性
- **实现**：
  - 第一层：使用轻量级模型进行快速筛选
  - 第二层：使用高精度模型进行精细排序

### 3. 上下文感知检索

- **方法**：考虑对话历史和上下文信息
- **优势**：适应多轮对话场景，提高上下文相关性
- **实现**：将历史对话作为检索的一部分

### 4. 知识图谱增强检索

- **方法**：结合知识图谱的结构化信息
- **优势**：利用实体关系，提高检索的准确性和可解释性
- **实现**：将知识图谱的结构信息融入向量表示

### 5. 动态检索参数

- **方法**：根据查询类型自动调整检索参数
- **优势**：适应不同类型的查询，提高整体性能
- **实现**：
  - 问题类型识别：识别查询是事实型、分析型还是开放型
  - 参数调整：根据查询类型调整 k 值、阈值等参数

## 检索策略的评估

### 评估指标

1. **召回率（Recall）**：检索到的相关文档占所有相关文档的比例
2. **精确率（Precision）**：检索到的文档中相关文档的比例
3. **F1 分数**：精确率和召回率的调和平均
4. **平均倒数排名（MRR）**：衡量第一个相关结果的排名
5. **nDCG**：归一化折损累积增益，考虑结果的排序质量

### 评估方法

1. **离线评估**：使用标注数据集进行评估
2. **在线评估**：在实际应用中收集用户反馈
3. **A/B 测试**：比较不同检索策略的性能

## 检索策略的优化

### 1. 向量优化

- **模型选择**：选择适合特定领域的 Embedding 模型
- **模型微调**：在领域数据上微调 Embedding 模型
- **向量压缩**：使用量化技术减少向量大小

### 2. 索引优化

- **索引类型**：选择合适的向量索引类型（如 HNSW、IVF）
- **索引参数**：调整索引参数以平衡速度和准确性
- **分区策略**：根据文档类型、时间等因素进行分区

### 3. 查询优化

- **查询重写**：对用户查询进行重写，提高检索效果
- **查询扩展**：添加相关术语，丰富查询语义
- **查询分类**：根据查询类型选择不同的检索策略

### 4. 结果处理

- **结果融合**：融合多个检索策略的结果
- **冗余过滤**：去除重复或高度相似的结果
- **多样性增强**：确保结果的多样性，避免信息重复

## 检索策略的应用场景

### 1. 通用问答

- **检索策略**：混合检索 + 重排序
- **特点**：需要平衡相关性和召回率
- **优化**：针对不同类型的问题调整检索参数

### 2. 专业领域问答

- **检索策略**：知识图谱增强检索 + 领域特定 Embedding
- **特点**：需要高精度和领域专业性
- **优化**：使用领域特定的知识库和模型

### 3. 多轮对话

- **检索策略**：上下文感知检索 + 会话历史融合
- **特点**：需要考虑对话的连贯性
- **优化**：维护会话状态，动态调整检索策略

### 4. 实时应用

- **检索策略**：分层检索 + 缓存机制
- **特点**：需要低延迟和高响应速度
- **优化**：使用轻量级模型和缓存技术

## 常见问题与解决方案

### 1. 检索结果不相关

**问题**：检索到的文档与查询相关性低
**解决方案**：
- 优化 Embedding 模型
- 使用混合检索策略
- 增加重排序步骤
- 调整检索参数

### 2. 检索速度慢

**问题**：检索过程耗时较长
**解决方案**：
- 使用更高效的索引类型
- 实现分层检索
- 优化向量存储
- 增加缓存机制

### 3. 上下文窗口限制

**问题**：检索到的文档超出模型上下文窗口
**解决方案**：
- 优化分块策略
- 实现结果摘要
- 使用层次化检索
- 动态调整检索结果数量

### 4. 冷启动问题

**问题**：知识库初期数据不足，检索效果差
**解决方案**：
- 预填充基础知识库
- 使用通用 Embedding 模型
- 逐步优化检索策略
- 收集用户反馈，持续改进
