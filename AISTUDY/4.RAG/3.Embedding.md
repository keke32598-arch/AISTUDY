# Embedding（文本向量化）

## 什么是 Embedding

Embedding 是将文本、图像、音频等非结构化数据转换为数值向量的过程。在 RAG 中，主要指文本向量化，即将文本转换为可以计算相似度的向量表示。

## Embedding 的作用

- **语义表示**：将文本的语义信息编码到向量中
- **相似度计算**：通过向量距离衡量文本间的语义相似度
- **降维处理**：将高维文本数据压缩到低维向量空间
- **模型输入**：为机器学习模型提供数值输入

## 常用的 Embedding 模型

### 1. OpenAI Embedding

- **模型**：text-embedding-3-small, text-embedding-3-large
- **维度**：1536（small）, 3072（large）
- **特点**：高质量、通用、API 调用
- **适用场景**：生产环境、需要高精度的场景
- **示例代码**：
  ```python
  import openai
  client = openai.OpenAI(api_key="your-key")
  response = client.embeddings.create(
      input="Hello world",
      model="text-embedding-3-small"
  )
  embedding = response.data[0].embedding
  ```

### 2. Hugging Face Embedding

- **模型**：BERT、Sentence-BERT、MiniLM 等
- **维度**：根据模型不同，通常 384-768
- **特点**：开源、可本地部署、可微调
- **适用场景**：自定义需求、隐私敏感场景
- **示例代码**：
  ```python
  from sentence_transformers import SentenceTransformer
  model = SentenceTransformer('all-MiniLM-L6-v2')
  embedding = model.encode("Hello world")
  ```

### 3. 国产大模型 Embedding

- **模型**：智谱 AI、讯飞星火、百度文心等
- **特点**：对中文支持更好
- **适用场景**：中文为主的应用

### 4. 开源轻量级模型

- **模型**：GTE、E5、BAAI/bge 等
- **特点**：轻量、快速、可本地部署
- **适用场景**：边缘设备、实时应用

## Embedding 模型的选择

### 选择因素

1. **语义质量**：向量的语义表示能力
2. **速度**：编码速度和推理时间
3. **维度**：向量维度（影响存储和计算）
4. **成本**：API 调用费用或计算资源
5. **语言支持**：对特定语言的处理能力
6. **部署方式**：云端 API 或本地部署

### 模型比较

| 模型 | 维度 | 速度 | 质量 | 成本 | 部署方式 |
|------|------|------|------|------|----------|
| OpenAI text-embedding-3-large | 3072 | 中 | 高 | 高 | 云端 API |
| Sentence-BERT | 768 | 中 | 中 | 低 | 本地 |
| MiniLM | 384 | 快 | 中 | 低 | 本地 |
| BAAI/bge | 768 | 中 | 高 | 低 | 本地 |

## Embedding 的最佳实践

### 输入处理

1. **文本清洗**：去除噪声、特殊字符
2. **长度控制**：根据模型限制调整文本长度
3. **格式统一**：保持输入格式一致
4. **批处理**：批量编码提高效率

### 向量存储

1. **维度选择**：平衡精度和存储成本
2. **数据类型**：使用合适的数值类型（如 float32）
3. **压缩策略**：考虑向量压缩技术
4. **索引优化**：选择合适的索引类型

### 性能优化

1. **缓存机制**：缓存重复文本的 embedding
2. **批量处理**：批量编码提高吞吐量
3. **并行计算**：利用多线程或 GPU 加速
4. **模型量化**：使用量化技术减少内存使用

## Embedding 的评估

### 评估指标

1. **语义相似度**：评估向量对语义相似文本的区分能力
2. **检索性能**：在 RAG 系统中评估检索效果
3. **计算效率**：编码速度和资源消耗
4. **存储成本**：向量存储空间

### 评估方法

1. **人工评估**：人工判断检索结果的相关性
2. **自动评估**：使用标准数据集进行评估
3. **在线评估**：在实际应用中收集用户反馈

## Embedding 的应用场景

- **文本检索**：语义搜索、相关文档查找
- **聚类分析**：文本聚类、主题发现
- **分类任务**：文本分类、情感分析
- **推荐系统**：基于内容的推荐
- **问答系统**：问题与答案的匹配

## 常见问题与解决方案

### 1. 长文本处理

**问题**：模型对长文本的处理能力有限
**解决方案**：
- 文本分块：将长文本分割成多个短文本
- 段落编码：对每个段落单独编码，然后聚合
- 摘要编码：先生成文本摘要，再对摘要编码

### 2. 多语言处理

**问题**：不同语言的 embedding 质量差异
**解决方案**：
- 使用多语言模型
- 针对特定语言选择专门的模型
- 考虑语言转换策略

### 3. 领域适应

**问题**：通用模型在特定领域表现不佳
**解决方案**：
- 领域微调：在领域数据上微调模型
- 领域特定模型：使用针对特定领域的模型
- 混合策略：结合通用模型和领域知识
